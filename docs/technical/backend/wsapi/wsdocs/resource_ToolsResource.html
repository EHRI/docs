POST /tools/regenerate-ids

Regenerate the hierarchical graph ID for a set of items, optionally renaming them.

The default mode is to output items whose IDs would change, without actually changing them. The collisions parameter will only output items that would cause collisions if renamed, whereas tolerant mode will skip them altogether.

The commit flag will cause renaming to take place.

Request Parameters
name type description default constraints multivalued
collisions query only output items that if renamed to the regenerated ID would cause collisions false boolean no
commit query whether or not to rename the item false boolean no
id query the existing item IDs     yes
tolerant query skip items that could cause collisions rather then throwing an error false boolean no
Request Body
media type data type
application/json (custom)
text/csv (custom)
Response Body
media type data type description
application/json (custom) a tab old-to-new mapping, or an empty body if nothing was changed

POST /tools/rename

Takes a CSV file containing two columns: the global id, and a new local identifier to rename an item to. A new global ID will be regenerated.

Input rows will be re-sorted lexically to ensure correct generation of dependent parent/child hierarchical IDs and output order will reflect this.
